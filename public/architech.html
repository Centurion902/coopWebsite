Job Description

My responsibilities as a software engineer at Architech included implementing conversation flows in a voice user interface (Google Home), creating cloud functions, prototyping data visualizations, and documenting my work. For each of these varied tasks I had to adapt and learn quickly as I had never worked with Google Home, or cloud services before. The technical skills for each job had to be picked on the job through research, experimentation and discussion with co-workers more knowledgeable of the technology. These included working with Dialogflow, Firebase, GCP and node. For the data visualization task though, my skills with javascript and css, as well as general programing principles learned during university projects are what helped me succeed.

Working with Dialogflow was definitely the most interesting part of my job simply because it required a different style of programing that was more state based than sequential. This made me completely rethink how I would have to structure my code in order to make the Google Home act as expected. Much of my work done at Architech was done with Googles cloud infrastructure, including writing cloud functions and using FireStore for scalable storage.

Documentation was a significant part of my job as well. To keep my projects maintainable, after I had completed a project I would document my findings and detail running and startup procedures so that others could work on the project after I had moved on.

Collaborating with others was also a key part of the job. Soft, interpersonal skills were key here that helped me effectively communicate with project managers, designers and other engineers in order to solve problems and clear up misunderstandings. These were especially useful at our end of sprint ceremonies where we would go over what went well and what could be improved on, as well as plan for the next sprint.

Cloud Computing and AutoML

For me, the focus of any working engagement is most often the technology I am working with. Working with Google cloud has put me in touch with the wide variety of machine learning based tools google has.

Dialogflow is the perfect jumping off point to talk about AutoML. Voice and phrase recognition is vital to the use of voice user interfaces (VUI's) and any developer of a VUI needs a machine learning tool that will do the hard work of learning the trigger phrases. Most developers are not machine learning engineers and this is where AutoML becomes the most useful. By setting up the pipeline for training, correcting and deploying the voice recognizing model, developers can now just give target phrases, let the pipeline train the model on google's hardware, and focus on developing their own applications. Developers that are unhappy with the achieved accuracy can add their own corrections to the training phrases and have models retrained in real time.

To pair with Dialogflow, I was working with Firebase which Google provides as a cloud service to host functions, databases websites and more. Its scalable serverless model was perfect for allowing us to prototype and develop quickly. When the service we were building was complete it would have no problem scaling up to match the number of users since the functions could have their instances multiplied.

I greatly enjoyed digging into this set of technologies and hope to work with similar ones soon.
